1.K8S基于HPA控制器实现Pod自动伸缩

HorizontalPodAutoscaler（简称HPA）自动更新工作负载资源（例如Deployment或者StatefulSet），基于pod资源利用率横向调整pod副本数量。目的是自动扩缩工作负载以满足需求。如果负载减少，并且Pod的数量高于配置的最小值， HorizontalPodAutoscaler会指示工作负载资源（Deployment、StatefulSet或其他类似资源）缩减。

三个版本之前的区别如下：

  • autoscaling/v1：只支持基于CPU指标的缩放；
  • autoscaling/v2beta1：支持 Resource Metrics（资源指标，如 pod 内存）和 Custom Metrics（自定义指标）的缩放；
  • autoscaling/v2beta2：支持 Resource Metrics（资源指标，如 pod 的内存）和 Custom Metrics（自定义指标）和 ExternalMetrics

1.1 安装metrics server

Metrics Server从node节点上的Kubelet收集资源指标,并通过Metrics API在 Kubernetes apiserver中公开指标数据,以供Horizontal Pod Autoscaler使用,也可以通过访问kubectl top node/pod查看指标数据。

github项目地址：https://github.com/kubernetes-sigs/metrics-server/releases

(4).Zabbix分布式监控Proxy

1.Zabbix分布式监控Proxy

Zabbix Proxy 可以代表 Zabbix server 收集性能和可用性数据。通过这种方式， proxy 可以自己承担一些收集数据的负载，并减轻 Zabbix
Server 的负担。

此外，当所有 agents 和 proxy 都向一个 Zabbix server 报告并且所有数据都集中收集时，使用 Proxy 代理是实现集中式和分布式监控的
最简单方法

  • Zabbix proxy 代理必须使用单独的数据库。
  • proxy 收集的所有数据在传输到 server 之前都存储在本地。这种方式不会因为与 server 之间的任何临时通信问题而丢失数据。
    ProxyLocalBuffer 和 ProxyOfflineBuffer 参数在proxy 配置文件控制数据在本地保存多长时间。
  • Zabbix proxy 代理是一个数据收集器。它不计算触发器、处理事件或发送警报。

1.3 配置Zabbix Proxy主动模式

配置并使用Zabbix Proxy主动模式收集zabbix agent监控数据。

主机名 IP地址 操作系统 服务
zabbix-server 192.168.1.30 Ubuntu20.04 Zabbix Server服务
zabbix-agent-active 192.168.1.33 Ubuntu20.04 Zabbix Agent服务(主动)
zabbix-proxy-active 192.168.1.35 Ubuntu20.04 Zabbix Proxy服务(主动)

管理->agent代理程序->创建代理

zabbix-agent-passive 192.168.1.34 Ubuntu20.04 Zabbix Agent服务(被动)

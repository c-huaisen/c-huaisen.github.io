(04).Docker构建镜像

1.Dockerfile

1.1 Dockerfile简介

​ Dockerfile是一个用来构建镜像的文本文件，由一行行命令语句组成，并且支持以 # 开头的注释行。一般的。Dockerfile分为四部分：基础镜像信息、维护者信息、镜像操作指令、容器启动时执行指令。

1.2 Dockerfile指令

1.2.1 FROM

FROM：指定构建新镜像时使用的基础镜像，通常必须是Dockerfile除了注释之外的第一行，但其前面也可以出现ARG指令。

1.2.2 RUN

RUN：基于FROM中定义的镜像为基础环境运行指定命令，并提交结果，生成的镜像将用于Dockerfile中的下一步。

1.2.3 CMD

CMD：为正在执行的容器提供默认值，默认值可以包括可执行文件，也可以省略可执行文件。Dockerfile中只能有一条CMD指令。如果列出多个CMD，则只有最后一个CMD生效。

1.2.4 LABEL

LABEL：将元数据添加到镜像。标签是键值对。

1.2.5 EXPOSE

EXPOSE：指定容器暴露的端口，支持TCP、UDP协议

1.2.6 ENV

ENV：设置容器环境变量

1.2.7 ADD

ADD：ADD指令从＜src＞复制新文件、目录或远程文件URL，并将它们添加到路径＜dest＞的映像文件系统中。会自动解压tar.gz格式的压缩包，不会自动解压zip。

1.2.8 COPY

COPY：从＜src＞复制新文件或目录，并将它们添加到路径＜dest＞的容器文件系统中。不会解压任何压缩。

1.2.9 ENTRYPOINT

ENTRYPOINT：用于定义容器在启动时候默认执行的命令或者脚本，如果是和CMD命令混合使用的时候，会将CMD的命令当做参数传递给ENTRYPOINT后面的脚本，可以在脚本中对参数做判断并相应的容器初始化操作。Dockerfile中只能有一条ENTRYPOINT指令。如果列出多个ENTRYPOINT，则只有最后一个ENTRYPOINT生效。

1.2.10 VOLUME

VOLUME：创建一个具有指定名称的挂载点，并将其标记为保存来自本机主机或其他容器的外部挂载卷。

1.2.11 USER

USER：

1.2.12 WORKDIR

WORKDIR：为Dockerfile中的任何RUN、CMD、ENTRYPOINT、COPY和ADD指令设置工作目录

1.2.13 ARG

ARG：定义一个变量。

1.2.14 ONBUILD

ONBUILD：特殊的指令，它后面跟的是其它指令，比如RUN``COPY等，而这些指令，在当前镜像构建时并不会被执行。只有当以当前镜像为基础镜像，去构建下一级镜像的时候才会被执行。
